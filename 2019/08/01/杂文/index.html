    前言

    为了让我的博客稍微能有点逼格，我选择了使用hexo来发布博客，以此文来记录这次搭建hexo的过程，但总的来说，这个方式比起简书，csdn等平台来说，发布起来要多一些步骤。但是好处是，没有广告，可以有丰富的主题，插件来diy。

    搭建

    搭建比较简单，直接用npm安装即可

    项目目录

    本文只大概讲解几个文件

    • hexo 主目录
      • public 将要被发布的目录，里面是处理好的html文件等
      • source 等待被处理的文件目录，里面有md文件
      • themes 主题
      • _config.yml 里面是配置，包括git地址等

    原理

    hexo通过对本地的由markdown书写的文档，进行预处理，按照主题生成html文件，然后通过git发布到 git pager上。然后通过浏览器就可以访问这些静态文件了。

    常用命令

    1. 帮助
      要是别的命令不记得就算了了，但是这个帮助命令要记得啦

      hexo help

    2. 新建
      这个命令会帮你在source文件夹下的_post文件夹下生成md文件，预写入时间等信息

      hexo new "filename"

    3. 生成
      这个命令会把你写好的文件处理成html放入public文件下

      hexo g 或者 hexo generate

    4. 发布
      这个命令就是把public文件夹发布到指定的git仓库里面去

      hexo d 或者 hexo deploye

    5. 删除
      这个命令可能不叫删除，但是删除要用到它，就叫他删除吧，删除流程是，先删除md，再clean,接着 g 和 d

      hexo clean
